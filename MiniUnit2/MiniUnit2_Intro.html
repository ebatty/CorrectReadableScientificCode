Writing clean scripts#

“Indeed, the ratio of time spent reading versus writing is well over 10 to 1. We are constantly reading old code as part of the effort to write new code. … Therefore, making it easy to read makes it easier to write.” – Robert C. Martin | Clean Code (p. 14)

In this mini-unit, we will focus on the essential clean code skill of writing readable scripts. We will leave functions and overall code base structure to future mini-units. Luckily, what you learn in this mini-unit about writing readable scripts will apply to any code you write, including code within functions.

Writing readable code will:

  • Save you time (quicker to understand what’s happening when you come back to code and make edits)

  • Save others time if they look at or use your code

  • Increase the likelihood of spotting errors, since the code is easier to reason about

During this mini-unit, you will assemble a toolkit of cleaning tools to clean your scripts and ensure they are readable.

The code below computes the absolute difference between two values.

Click here for solution explanation

There are a few things making this snippet less readable:

  • Unclear variable names

  • Having everything on one line

  • Not obeying Python style guide conventions for whitespaces around operators

  • The else clause isn’t necessary if we use >= or <=

We can fix these issues with:

a = 10
b = 4

# Computes the absolute difference
if a >= b:
  absolute_difference = a - b
elif a < b:
  absolute_difference = b - a

Even better though, we could use the absolute value function that already exists in Python! In general, you should not re-implement common computations, so if you see something like this, you should google “how to get absolute difference in Python”

a = 10
b = 4

# Computes the absolute difference
absolute_difference = abs(a - b)

B) Diving into DNA.

The code below translates a sequence of DNA nucleotides

DNA consists of a sequence of nucleotides (A, G, T, or C). In the genetic code, each group of three consecutive nucleotides form a codon that translates to a single amino acid. There are a small number of common amino acids - we can use a look-up table to pair each codon with the respective amino acid. We also have start codons, which indicate that we should start translating after it appears (so starting with the following codon) and stop codons that signify that the DNA should stop being translated there. For our purposes here, we can assume the dna sequence just has one start and one stop codon.

In this way, we can work through a DNA sequence, taking each group of three nucleotides (the first three, then the next three, and so on) and translating them to the corresponding amino acid. The resulting sequence of amino acids constitutes the protein.

C) Neural tuning to reach angle

Please note this data is simulated for exercise purposes

We have recordings from 9 different motor cortex neurons from a macaque. We have recorded these neurons while the macaque is reaching to 8 different directions.

  • The spiking data array spikes is 9 by 1000 by 1500. The first dimension corresponds to different neurons, the second to 1000 different trials, and the third to 1500 time bins of width 1 ms each. If the neuron spikes during that time bin on that trial, the entry is 1, otherwise it is 0. The reach movement onset occurs at the 750th time bin.

  • The angles_by_trial array is shape (1000,). It contains the reaching angle on each trial - which is one of 8 unique reach angles.

In the code below, we look at how the firing rate of neuron 2 varies by angle. For a given angle, we compute the mean firing rate in spikes per second for each reach angle. We get this mean firing rate from considering the 500th to 1000th time bin (so the chunk of time around reach onset) over all trials of that reach angle. In the code, we do this by summing up the spikes in that time chunk on all trials of that reach angle and divide by the number of trials of that reach angle and the length of time per trial (0.5 seconds).

We then plot the average firing rate per angle vs the angle, and find the angle that lead to the highest firing rate. As you clean up the code, the plot should not change. To make it easier to compare to the correct plot, we include an image of the correct plot below the code.

n_neurons = 8
n_trials = 1000
n_time_bins_per_trial = 1500
bin_width = 0.001 # seconds
reaching_angles = np.array([0, 45, 90, 135, 180, 225, 270, 315])
n_angles = reaching_angles.shape[0]

# Generate even number of trials for each angle, then shuffle
trial_angles = np.repeat(reaching_angles, int(1000/8))
np.random.shuffle(trial_angles)


spikes = np.zeros((n_neurons, n_trials, n_time_bins_per_trial))

baseline_firing_rate = 15
firing_rates_per_angle = np.array([5, 8, 40, 54, 55, 38, 24, 8])
binned_spikes = np.zeros((n_trials, ))
for i_trial in range(n_trials):
    reach_firing_rates = 15*np.ones((1500))
    angle_pos = np.where(reaching_angles == trial_angles[i_trial])[0]
    reach_firing_rates[(750 - 250):(750 + 250)] = firing_rates_per_angle[angle_pos]
    spikes[1, i_trial, :] = np.random.poisson(reach_firing_rates * bin_width)

import numpy as np
import matplotlib.pyplot as plt

# Compute average firing rate per angle
avfr = np.zeros((8,))
for i in range(8): # loops over angles
   ss = 0 # tracks sum of spikes
   for j in range(1000): # loops over trials
      if trial_angles[j]==reaching_angles[i]: # checks if that trial was that angle
          for t in range(500,1000): # loop over relevant time bins
